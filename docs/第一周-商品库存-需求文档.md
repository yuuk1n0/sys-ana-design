# 第一周商品/库存模块需求文档（PRD）

## 1. 文档目标

明确第一周交付的业务需求、功能范围、角色权限、接口范围和验收标准，支撑“商品管理 + 门店库存”上线演示。

## 2. 业务背景

当前阶段聚焦店长对个人门店的基础运营能力建设。门店日常运营中，商品信息准确性和库存可见性是收银与后续盘点的前置条件，因此第一周优先落地商品与库存模块。

## 3. 本期范围（W1）

## 3.1 包含范围

- 商品分类管理
- 商品管理（建档、编辑、上下架、状态）
- 门店库存余额查询
- 门店库存流水查询
- 低库存预警
- 门店级数据隔离

## 3.2 不包含范围

- 收银开单与支付
- 退货业务
- 盘点差异处理
- 日结关账
- 采购入库与门店调拨

## 4. 用户角色与权限

- 店长
  - 可访问商品分类、商品管理、库存余额、库存流水、库存预警
  - 可执行商品新增/编辑/上下架
- 收银员
  - 本周仅可查看商品列表与库存余额（若开放）
  - 不可执行商品维护操作

权限规则：
- 所有列表与详情接口按 `store_id` 自动过滤
- 非本门店数据一律不可见、不可操作

## 5. 功能需求

## 5.1 商品分类管理

需求说明：
- 店长可维护门店商品分类

功能点：
- 分类列表查询
- 新增分类
- 编辑分类
- 删除分类

业务规则：
- 同门店内分类名称唯一
- 被商品引用的分类不可删除

## 5.2 商品管理

需求说明：
- 店长可管理商品基础档案，支持后续收银检索与库存关联

功能点：
- 商品分页查询（名称、分类、状态筛选）
- 商品创建
- 商品编辑
- 商品上下架

业务规则：
- 商品编码在门店内唯一
- 条码非空时在门店内唯一
- 下架商品保留历史数据，但不允许后续销售使用
- 商品创建后自动生成库存余额记录

## 5.3 库存余额

需求说明：
- 店长可查看商品当前库存与预警状态

功能点：
- 库存余额分页查询
- 按商品名称/分类筛选
- 低库存高亮展示

业务规则：
- 低库存条件：`available_qty <= low_stock_threshold`

## 5.4 库存流水

需求说明：
- 店长可追溯库存变化来源

功能点：
- 流水分页查询
- 按商品、业务类型、时间区间筛选

业务规则：
- 每次库存变更必须落一条流水
- 流水记录不可修改、不可删除

## 5.5 低库存预警

需求说明：
- 快速识别补货风险商品

功能点：
- 仅展示低库存商品清单

业务规则：
- 预警逻辑实时依据库存余额计算

## 6. API 范围（W1）

后端接口分组：
- 商品分类：4个接口（list/create/update/delete）
- 商品：5个接口（list/get/create/update/change_status）
- 库存：3个接口（balance_list/txn_list/warning_list）

前端集成要求：
- 页面列表全部使用分页接口
- 查询条件与分页参数保持 URL 可回放
- 所有操作成功后刷新当前页数据

## 7. 页面与交互需求

## 7.1 商品分类页

- 列表 + 弹窗 CRUD
- 删除前二次确认

## 7.2 商品管理页

- 查询区：名称、分类、状态
- 表格区：基础字段 + 库存摘要
- 弹窗区：创建/编辑表单
- 行操作：编辑、上架/下架

## 7.3 库存余额页

- 查询区：商品名称、分类
- 表格区：可用库存、预警阈值、库存状态

## 7.4 库存流水页

- 查询区：商品、业务类型、时间区间
- 表格区：变更前后数量、变更量、业务单号、操作人、时间

## 7.5 库存预警页

- 只展示低库存商品
- 支持跳转商品详情（可选）

## 8. 非功能需求

- 性能：常用列表接口 P95 < 1 秒（课程规模数据）
- 安全：JWT + RBAC；关键操作写审计日志
- 一致性：商品创建与库存初始化在同一事务内
- 可维护性：遵循现有 `app/api/v1 + controller + model + schema` 结构

## 9. 数据需求

最小数据实体：
- `product_category`
- `product`
- `store_inventory`
- `inventory_txn`

关键约束：
- 商品编码唯一约束
- 条码唯一约束（非空）
- 门店+商品库存唯一约束

## 10. 里程碑拆分（W1）

- D1-D2：表结构落地 + 商品分类/商品 API
- D3-D4：库存余额/流水/预警 API
- D5：前端页面联调 + 权限配置 + 验收

## 11. 验收标准

- 店长可完成分类与商品维护
- 商品创建后可在库存余额中查询到初始化记录
- 库存流水支持按条件查询并可追溯
- 低库存预警页可正确显示风险商品
- 门店数据隔离生效，越权访问被拒绝

## 12. 风险与应对

- 风险：商品与库存初始化不一致
  - 应对：创建商品与初始化库存使用事务
- 风险：门店越权查询
  - 应对：服务端统一强制注入 `store_id`
- 风险：条码重复导致收银检索冲突
  - 应对：数据库唯一约束 + 接口前置校验
