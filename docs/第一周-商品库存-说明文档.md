# 第一周商品/库存模块说明文档（技术方案）

## 1. 文档目的

本说明文档用于定义第一周交付范围内的技术实现方案，覆盖：
- 后端表结构设计
- 商品/库存 API 设计
- 商品/库存前端页面设计
- 权限菜单与数据隔离策略

对应第一阶段范围：仅实现店长个人门店的商品与库存管理基础能力。

## 2. 第一周交付边界

本周包含：
- 商品分类管理
- 商品管理（基础档案 + 状态）
- 门店库存余额查询
- 门店库存流水查询
- 低库存预警查询

本周不包含：
- POS 开单与支付
- 退货、盘点、日结
- 采购与调拨

## 3. 后端表结构设计

## 3.1 命名与通用约束

- 所有业务表包含：`id`、`store_id`、`created_at`、`updated_at`
- 所有查询接口默认按 `store_id` 过滤
- `store_id + code` 或 `store_id + name` 组合索引优先

## 3.2 表清单（W1）

1) `product_category`（商品分类）
- 用途：维护门店可用分类
- 索引：`idx_product_category_store_name(store_id, name)`

2) `product`（商品主档）
- 用途：维护商品基础信息
- 索引：
  - `uk_product_store_code(store_id, product_code)`
  - `uk_product_store_barcode(store_id, barcode)`（允许空值）
  - `idx_product_store_status(store_id, status)`

3) `store_inventory`（门店库存余额）
- 用途：保存每个商品当前库存
- 索引：
  - `uk_inventory_store_product(store_id, product_id)`
  - `idx_inventory_store_available(store_id, available_qty)`

4) `inventory_txn`（库存流水）
- 用途：记录库存变化轨迹
- 索引：
  - `idx_txn_store_product_time(store_id, product_id, created_at)`
  - `idx_txn_store_type_time(store_id, biz_type, created_at)`

## 3.3 字段定义（建议 SQL）

```sql
CREATE TABLE product_category (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  store_id BIGINT NOT NULL,
  name VARCHAR(64) NOT NULL,
  sort INT NOT NULL DEFAULT 0,
  status TINYINT NOT NULL DEFAULT 1,
  created_at DATETIME NOT NULL,
  updated_at DATETIME NOT NULL,
  UNIQUE KEY uk_category_store_name (store_id, name),
  KEY idx_category_store_status (store_id, status)
);

CREATE TABLE product (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  store_id BIGINT NOT NULL,
  category_id BIGINT NOT NULL,
  product_code VARCHAR(64) NOT NULL,
  name VARCHAR(128) NOT NULL,
  barcode VARCHAR(64) NULL,
  unit VARCHAR(16) NOT NULL,
  sale_price DECIMAL(10,2) NOT NULL,
  status TINYINT NOT NULL DEFAULT 1,
  stock_status TINYINT NOT NULL DEFAULT 1,
  low_stock_threshold INT NOT NULL DEFAULT 0,
  remark VARCHAR(255) NULL,
  created_at DATETIME NOT NULL,
  updated_at DATETIME NOT NULL,
  UNIQUE KEY uk_product_store_code (store_id, product_code),
  UNIQUE KEY uk_product_store_barcode (store_id, barcode),
  KEY idx_product_store_status (store_id, status),
  KEY idx_product_store_category (store_id, category_id)
);

CREATE TABLE store_inventory (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  store_id BIGINT NOT NULL,
  product_id BIGINT NOT NULL,
  available_qty INT NOT NULL DEFAULT 0,
  locked_qty INT NOT NULL DEFAULT 0,
  low_stock_threshold INT NOT NULL DEFAULT 0,
  version INT NOT NULL DEFAULT 1,
  updated_by BIGINT NULL,
  created_at DATETIME NOT NULL,
  updated_at DATETIME NOT NULL,
  UNIQUE KEY uk_inventory_store_product (store_id, product_id),
  KEY idx_inventory_store_available (store_id, available_qty)
);

CREATE TABLE inventory_txn (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  store_id BIGINT NOT NULL,
  product_id BIGINT NOT NULL,
  biz_type VARCHAR(32) NOT NULL,
  biz_no VARCHAR(64) NOT NULL,
  change_qty INT NOT NULL,
  before_qty INT NOT NULL,
  after_qty INT NOT NULL,
  remark VARCHAR(255) NULL,
  operator_id BIGINT NULL,
  created_at DATETIME NOT NULL,
  updated_at DATETIME NOT NULL,
  KEY idx_txn_store_product_time (store_id, product_id, created_at),
  KEY idx_txn_store_type_time (store_id, biz_type, created_at)
);
```

字段枚举约定：
- `product.status`：`1=上架`，`0=下架`
- `product.stock_status`：`1=有货`，`0=缺货`
- `inventory_txn.biz_type`：`INIT`、`SALE`、`RETURN`、`STOCKTAKE_GAIN`、`STOCKTAKE_LOSS`、`MANUAL`

## 4. 后端 API 设计（W1）

统一前缀：`/api/v1`

响应结构沿用当前项目：
- 成功：`{ code: 200, msg: "OK", data: ... }`
- 列表：`{ code: 200, data: [...], total, page, page_size }`

## 4.1 商品分类 API

1) `GET /product-category/list`
- 入参：`name?`、`status?`
- 出参：分类列表

2) `POST /product-category/create`
- 入参：`name`、`sort`、`status`
- 校验：同门店分类名称唯一

3) `POST /product-category/update`
- 入参：`id`、`name?`、`sort?`、`status?`

4) `DELETE /product-category/delete`
- 入参：`id`
- 校验：分类下存在商品时禁止删除

## 4.2 商品 API

1) `GET /product/list`
- 入参：`page`、`page_size`、`name?`、`category_id?`、`status?`、`stock_status?`
- 出参：商品分页列表（含库存摘要：`available_qty`）

2) `GET /product/get`
- 入参：`id`
- 出参：商品详情

3) `POST /product/create`
- 入参：`category_id`、`product_code`、`name`、`unit`、`sale_price`、`barcode?`、`low_stock_threshold?`、`status`
- 规则：`product_code` 门店唯一；`barcode` 非空时门店唯一
- 动作：自动初始化 `store_inventory` 与 `inventory_txn(INIT)`

4) `POST /product/update`
- 入参：`id` + 可更新字段
- 规则：下架商品仍可查询，不可用于后续销售

5) `POST /product/change_status`
- 入参：`id`、`status`
- 规则：状态变更实时影响收银可售校验

## 4.3 库存 API

1) `GET /inventory/balance/list`
- 入参：`page`、`page_size`、`name?`、`category_id?`、`stock_status?`
- 出参：库存余额分页，字段含 `available_qty`、`low_stock_threshold`、`is_low_stock`

2) `GET /inventory/txn/list`
- 入参：`page`、`page_size`、`product_id?`、`biz_type?`、`start_time?`、`end_time?`
- 出参：库存流水分页

3) `GET /inventory/warning/list`
- 入参：`page`、`page_size`
- 出参：低库存商品分页（`available_qty <= low_stock_threshold`）

## 4.4 错误码建议

- `40001` 参数校验失败
- `40002` 分类名称重复
- `40003` 商品编码重复
- `40004` 商品条码重复
- `40005` 分类仍被商品引用
- `40006` 商品不存在或无门店权限

## 5. 前端页面设计（W1）

## 5.1 页面清单

1) 商品分类页：`/store/product-category`
- 功能：分类列表、创建、编辑、删除

2) 商品管理页：`/store/product`
- 功能：商品分页、筛选、创建、编辑、上/下架

3) 库存余额页：`/store/inventory/balance`
- 功能：库存分页、低库存高亮

4) 库存流水页：`/store/inventory/txn`
- 功能：流水分页、按商品/类型/时间筛选

5) 低库存预警页：`/store/inventory/warning`
- 功能：仅展示预警商品

## 5.2 页面字段与交互要点

- 商品管理列表字段：
  - 商品编码、名称、分类、单位、售价、状态、有货/缺货、可用库存、更新时间
- 商品编辑弹窗字段：
  - 分类、编码、名称、条码、单位、售价、预警阈值、上下架状态
- 库存余额列表字段：
  - 商品、分类、可用库存、预警阈值、库存状态（正常/低库存）
- 库存流水列表字段：
  - 时间、商品、业务类型、业务单号、变更量、变更前、变更后、操作人

## 5.3 前端 API 封装建议

在 `web/src/api/index.js` 增加：
- `getProductCategoryList / createProductCategory / updateProductCategory / deleteProductCategory`
- `getProductList / getProductById / createProduct / updateProduct / changeProductStatus`
- `getInventoryBalanceList / getInventoryTxnList / getInventoryWarningList`

## 7. 第一周验收检查单

- 可完成商品分类与商品建档
- 商品建档后自动生成库存余额与初始化流水
- 商品列表可按名称/分类筛选
- 库存余额、库存流水、低库存预警均可分页查询
- 不同门店账号只能看到本店数据
