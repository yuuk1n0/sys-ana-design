# 连锁超市管理系统需求文档（PRD）

## 1. 项目背景

基于现有 `vue-fastapi-admin` 项目，建设一个可用于课程作业答辩的连锁超市管理系统。系统按三部分推进，当前只交付第一部分：店长管理个人门店所需的核心经营能力。

## 2. 项目目标

- 形成“商品管理 -> 库存管理 -> 收银销售 -> 盘点调整 -> 日结归档”的单门店闭环
- 支持店长对个人门店的独立经营管理
- 保证关键数据可追溯、可核对、可审计
- 为后续采购、调拨、总部分析预留扩展空间

## 3. 三部分拆分范围

### 3.1 第一部分（本期）

- 商品管理
- 门店库存
- 收银（POS）
- 盘点
- 日结

### 3.2 第二部分（后续）

- 采购入库
- 门店调拨
- 供应商协同

### 3.3 第三部分（后续）

- 结算对账深化
- 总部经营分析
- 多门店协同优化

## 4. 本期角色与权限

- **店长**
  - 管理个人门店商品、库存、盘点、日结
  - 查看个人门店销售数据
- **收银员**
  - 执行销售商品的crud操作，影响商品库存和销售统计

权限约束：
- 全部业务数据按 `store_id` 过滤
- 店长仅可访问所属门店数据
- 关键操作必须记录审计日志

## 5. 本期功能需求（第一部分）

### 5.1 商品管理模块（高优先级）

#### 功能点
- 商品基础档案管理：名称、分类、单位、售价、状态
- 门店商品状态管理：有货/缺货
- 商品查询：按名称、分类检索

#### 关键规则
- 商品编码或条码在门店范围内唯一
- 停售商品不可新增销售单明细
- 商品状态变更需即时作用于收银端

#### 验收标准
- 店长可完成商品新增、编辑、上下架
- 收银端可按条码检索并识别商品状态

### 5.2 门店库存模块（高优先级）

#### 功能点
- 库存余额查询：按商品查看可用库存
- 库存流水查询：销售、退货、盘点调整
- 库存预警：低库存提醒

#### 关键规则
- 所有库存变化必须写入流水
- 默认不允许负库存销售
- 库存调整必须关联业务单据或盘点单

#### 验收标准
- 销售与退货后库存自动变化
- 店长可按商品追溯库存变动来源

### 5.3 收银（POS）模块（高优先级）

#### 功能点
- 开单：扫码/检索商品、修改数量、计算金额
- 支付：记录支付方式与支付金额
- 退货：按原销售单执行退货

#### 关键规则
- 支付成功后订单状态置为已完成并扣减库存
- 退货数量不得超过原单可退数量
- 收银异常中断不得产生脏订单

#### 验收标准
- 收银员可完成完整销售流程
- 退货后库存与销售统计同步更新

### 5.4 盘点模块（高优先级）

#### 功能点
- 盘点单创建与商品范围选择
- 实盘数录入与差异计算
- 盘点确认并更新库存

#### 关键规则
- 盘点确认前不得影响现有库存
- 盘盈盘亏处理必须生成库存流水
- 盘点确认权限仅限店长

#### 验收标准
- 可生成盘点差异并一键确认入账
- 可查询历史盘点单与差异明细

### 5.5 日结模块（高优先级）

#### 功能点
- 日结汇总：订单数、销售额、退款额、支付方式汇总
- 班次汇总：按收银员或班次查看
- 关账确认：完成营业日归档

#### 关键规则
- 每个门店每天仅允许一次最终关账
- 关账后当日汇总口径锁定
- 关账需保留操作人和时间戳

#### 验收标准
- 店长可在当日营业结束后完成关账
- 可追溯任一天日结结果与构成明细

## 6. 非功能需求

- **性能**：门店场景下常用查询接口响应时间 < 1s
- **安全**：JWT + RBAC，关键操作全量审计
- **可用性**：核心模块可在主流浏览器稳定运行
- **可维护性**：遵循现有项目编码结构与命名规范

## 7. 数据要求

- 核心业务表均包含 `store_id`
- 库存变更必须有 `inventory_txn` 记录
- 日结数据可按“门店 + 日期”唯一定位

## 8. 本期不实现范围

- 采购入库流程与供应商协同
- 门店调拨流程
- 对公结算、财务对账、复杂利润核算

## 9. 里程碑计划（本期）

- **W1**
  - 商品管理、库存查询、基础权限过滤
- **W2**
  - POS 开单、支付、退货、库存联动
- **W3**
  - 盘点流程、日结流程、报表汇总
- **W4**
  - 联调测试、验收演示数据准备

## 10. 交付物清单（本期）

- 后端 API 与数据模型
- 前端管理页面与权限菜单
- 店长与收银员角色权限配置
- 测试清单与演示脚本
- 说明文档与本需求文档

## 11. 验收标准（本期）

- 完成端到端闭环：商品建档 -> 销售 -> 库存变化 -> 盘点 -> 日结
- 店长账号仅可访问个人门店数据
- 收银员账号不可执行盘点确认与日结关账
- 任意库存变化可追溯到销售/退货/盘点单据
- 系统演示过程中无阻断性错误
