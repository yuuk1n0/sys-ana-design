# 第一阶段：超市管理系统数据建模成果汇报（PPT讲稿）

## 第1页 封面
- 题目：超市管理系统第一期逻辑数据建模
- 副标题：代码 + ORM 方式实现实体关系模型
- 团队/姓名/日期

---

## 第2页 项目背景与目标
- 第一阶段目标：形成“商品 -> 销售 -> 库存 -> 日结”闭环
- 建模目标：
  - 支撑一期功能上线
  - 满足后续二期扩展
  - 保障数据可追溯、可审计、可维护

---

## 第3页 建模方法与技术栈
- 建模方法：逻辑数据模型（实体、关系、约束）
- 实现方式：`Python 实体类 + Tortoise ORM`
- 技术栈：`FastAPI + Tortoise ORM + MySQL`
- 参考风格：实体类集中定义，业务域分组建模

---

## 第4页 一期业务域拆分
- 商品管理域：分类、商品主数据
- 会员与员工域：会员管理、岗位管理
- 销售域：订单、明细、支付、退货
- 仓储域：仓库库存、库存流水
- 财务域：日结汇总、支付方式汇总
- 系统域：角色、权限、员工角色绑定

---

## 第5页 抽象层次关系（泛化）
- 抽象父类：`Party`
- 子类实体：`Member`、`Employee`、`Supplier`
- 设计收益：
  - 共性字段复用（编码、名称、联系方式、状态）
  - 降低重复建模成本
  - 提升后续扩展一致性

---

## 第6页 量词关系（基数）设计
- 1:N：
  - `Store -> SalesOrder`
  - `SalesOrder -> SalesOrderItem`
  - `SalesOrder -> PaymentRecord`
  - `Warehouse -> WarehouseStock`
- N:N：
  - `Product <-> Supplier`
  - `Employee <-> RoleTemplate`
  - `RoleTemplate <-> Permission`
- 唯一约束：
  - 门店+营业日唯一日结
  - 仓库+商品唯一库存行

---

## 第7页 核心ER图展示
- 展示内容：
  - 主链路：商品、订单、库存、日结
  - 扩展链路：采购与供应关系
  - 权限链路：员工-角色-权限
- 图建议：
  - 使用 Mermaid 或 UML 工具输出
  - 在答辩中重点讲清“主键/外键/唯一约束”

---

## 第8页 关键数据规则落地
- 商品编码与条码唯一
- 每日每店唯一关账记录
- 库存变化必须记录流水（业务单号、前后数量）
- 销售、退货、支付、日结可形成闭环追溯
